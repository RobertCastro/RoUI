Cómo usarlo

Opción A · Un solo archivo (cualquier proyecto)

<link rel="stylesheet" href="dist/roui.css"> <body class="ro-root"> … </body>

Opción B · Con bundler (Vite / PostCSS / Next)

import '@robertcastro/roui/tokens.css'; import '@robertcastro/roui/component-tokens.css'; import '@robertcastro/roui/components/button.css';

Opción C · Tailwind

// tailwind.config.js presets: [require('roui/tailwind')]

Quality gates locales

Antes de integrar un cambio, la misma puerta de calidad se ejecuta localmente y en CI.

nvm use 20 npm ci npm run validate

Incluye generación DTCG, contraste, inventario de literales, build, lint, tarball, consumidores y presupuesto de tamaño.

Arquitectura

Estructura modular tipo Tailwind: tokens separados de componentes, y un bundle plano generado.

src/
├─ tokens/      DTCG source · CSS generado · temas · roles de componente
├─ base/        reset
├─ layouts/     shell · grid · three-col
├─ components/  button · badge · card · form · nav · header · sidebar · …
├─ utilities/   helpers
└─ index.css    @import de todo  ──build──▶  dist/roui.css
tokens/         tokens.json · tailwind.preset.cjs
docs/           esta galería

Regenerar el bundle: npm run build

Principios

Tokens primero

Todo color/tamaño es una variable --ro-*. Re-tematizas cambiando tokens, no componentes.

Portable

CSS plano con prefijo ro-. Sin dependencias, sin colisiones, funciona en cualquier framework.

Roles semánticos

--ro-bg, --ro-surface, --ro-accent… apuntan a primitivos y son fáciles de remapear.
