Overlays

Modal

experimental

Diálogo modal que interrumpe el flujo para una tarea o confirmación, con foco atrapado.

Cuándo usarlo

  • Confirmar una acción importante o destructiva.
  • Pedir un dato imprescindible antes de continuar.

Cuándo no

  • Para mensajes no bloqueantes: usa Alert o Toast.
  • Para contenido lateral navegable: usa Drawer.

Anatomía

<button data-ro-overlay-open="m1" aria-haspopup="dialog" aria-controls="m1" aria-expanded="false">Abrir</button>
<div class="ro-overlay" id="m1" data-ro-overlay-root hidden>
  <div class="ro-modal" role="dialog" aria-modal="true" aria-labelledby="m1-t" tabindex="-1">
    <div class="ro-modal__header">
      <h3 class="ro-modal__title" id="m1-t">Título</h3>
      <button class="ro-btn-icon" data-ro-overlay-close aria-label="Cerrar"><svg class="ro-icon" aria-hidden="true"><use href="#ro-i-x"></use></svg></button>
    </div>
    <div class="ro-modal__body">Contenido…</div>
  </div>
</div>
  • El root (data-ro-overlay-root) inicia con hidden y cubre la página.
  • El panel es role=dialog aria-modal=true, con nombre por aria-labelledby y tabindex=-1.
  • El disparador declara aria-haspopup=dialog y aria-controls; el controlador togglea aria-expanded.
  • Los controles de cierre llevan data-ro-overlay-close.

API

Clases

ClaseDescripción
.ro-overlayCapa que cubre la página (backdrop).
.ro-modalPanel del diálogo.
.ro-modal__headerCabecera con título y cierre.
.ro-modal__titleTítulo del diálogo (aria-labelledby).
.ro-modal__bodyCuerpo del diálogo.
.ro-modal__footerPie con acciones.

Atributos data

AtributoDescripción
data-ro-overlay-rootMarca el contenedor del overlay (inicia hidden).
data-ro-overlay-openEn el disparador: id del overlay a abrir.
data-ro-overlay-closeEn un control: lo cierra.
data-ro-overlay-backdropZona de fondo que cierra al hacer clic (opcional).

Tokens que consume

--ro-radius-2xl --ro-shadow-md --ro-z-modal --ro-white

Primitiva

import { createOverlayController } from "@robertcastro/roui/primitives/overlay-controller";

Teclado y accesibilidad

TeclaAcción
Tab / Shift+TabCicla el foco dentro del diálogo (trap).
EscapeCierra el diálogo y restaura el foco al disparador.
Enter / EspacioActiva el control enfocado.

Contrato de accesibilidad: dialog-drawer

  • Al abrir, el foco entra al diálogo; al cerrar, vuelve al disparador.
  • El fondo queda inerte: no se puede tabular fuera del diálogo.
  • Se bloquea el scroll del cuerpo mientras está abierto.

Do / Don't

Sí

  • Da un nombre accesible al diálogo con aria-labelledby.
  • Devuelve el foco al disparador al cerrar.
  • Ofrece siempre una salida clara (Escape y botón de cierre).

No

  • No abras un modal sin motivo que interrumpa la tarea.
  • No dejes foco alcanzable detrás del diálogo.
  • No anides un modal dentro de otro.
