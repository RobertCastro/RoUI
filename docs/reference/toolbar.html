Toolbar

experimental

Barra que agrupa acciones y controles de una vista, con separadores y grupos.

Cuándo usarlo

  • Reunir las acciones de una tabla, editor o panel en una sola barra.
  • Separar visualmente grupos de controles relacionados.

Cuándo no

  • Para la navegación principal del sitio: usa un landmark nav.
  • Para una sola acción aislada: un botón suelto basta.

Anatomía

<div class="ro-toolbar" role="toolbar" aria-label="Acciones de la tabla">
  <span class="ro-toolbar__title">12 seleccionados</span>
  <div class="ro-toolbar__group">
    <button class="ro-btn ro-btn--soft ro-btn--sm">Exportar</button>
    <span class="ro-toolbar__sep"></span>
    <button class="ro-btn ro-btn--soft ro-btn--sm">Eliminar</button>
  </div>
</div>
  • El contenedor lleva role=toolbar y un nombre accesible (aria-label).
  • ro-toolbar__sep es decorativo: no debe recibir foco ni anunciarse.
  • Los controles internos siguen siendo elementos nativos enfocables.

API

Clases

ClaseDescripción
.ro-toolbarBarra contenedora.
.ro-toolbar--flushSin fondo ni borde (embebida).
.ro-toolbar__groupAgrupa controles relacionados.
.ro-toolbar__sepSeparador visual entre grupos.
.ro-toolbar__titleRótulo o contexto de la barra.

Tokens que consume

--ro-radius-xl --ro-ink-10 --ro-space-2 --ro-white

Sin JavaScript: es CSS sobre HTML nativo.

Teclado y accesibilidad

TeclaAcción
Tab / Shift+TabRecorre los controles de la barra.
Enter / EspacioActiva el control enfocado.

Contrato de accesibilidad: native-controls

  • role=toolbar agrupa semánticamente; requiere aria-label.
  • El patrón APG con flechas puede añadirse cuando la barra sea densa; por ahora usa foco por Tab.

Do / Don't

Sí

  • Da a la barra un nombre accesible con aria-label.
  • Agrupa acciones afines y sepáralas con ro-toolbar__sep.

No

  • No metas la navegación del sitio en una toolbar.
  • No hagas enfocable el separador.

Madurez

experimental Apariencia o API sin garantías completas; puede cambiar sin aviso.
