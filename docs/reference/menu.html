Menu

experimental

Menú flotante de acciones u opciones, abierto desde un disparador (dropdown).

Cuándo usarlo

  • Agrupar acciones secundarias de un elemento (kebab, perfil).
  • Ofrecer opciones excluyentes marcadas con aria-current.

Cuándo no

  • Para navegación principal del sitio: usa un nav.
  • Para seleccionar de una lista larga con filtro: usa Combobox.

Anatomía

<div class="ro-dropdown" data-ro-disclosure-root>
  <button data-ro-disclosure-trigger aria-haspopup="menu" aria-controls="m1" aria-expanded="false">Opciones</button>
  <div class="ro-menu" id="m1" data-ro-disclosure-panel role="menu" aria-label="Opciones" hidden>
    <button class="ro-menu__item" role="menuitem">Editar</button>
    <button class="ro-menu__item ro-menu__item--danger" role="menuitem">Eliminar</button>
  </div>
</div>
  • El disparador declara aria-haspopup=menu y aria-expanded.
  • El panel es role=menu con nombre; los ítems son role=menuitem.
  • Los rótulos internos usan role=presentation; el separador es decorativo.

API

Clases

ClaseDescripción
.ro-dropdownContenedor posicionador.
.ro-menuPanel del menú (role=menu).
.ro-menu--endAlinea el menú a la derecha.
.ro-menu__itemOpción del menú (role=menuitem).
.ro-menu__item--dangerAcción destructiva.
.ro-menu__labelRótulo de grupo (role=presentation).
.ro-menu__sepSeparador decorativo.

Atributos data

AtributoDescripción
data-ro-disclosure-rootContenedor disparador + panel.
data-ro-disclosure-triggerBotón que abre/cierra.
data-ro-disclosure-panelPanel del menú.

Tokens que consume

--ro-radius-xl --ro-shadow-md --ro-z-overlay --ro-white

Primitiva

import { createDisclosureController } from "@robertcastro/roui/primitives/disclosure-controller";

Teclado y accesibilidad

TeclaAcción
Enter / EspacioAbre y cierra el menú desde el disparador.
EscapeCierra el menú y devuelve el foco al disparador.
TabRecorre los ítems mientras está abierto.

Contrato de accesibilidad: menu-popover-tooltip

  • Reserva role=menu para listas de acciones, no para navegación.
  • Marca la opción activa con aria-current, no con una clase de estado.

Ejemplos

Menú de acciones (interactivo)
<div class="ro-dropdown" data-ro-disclosure-root>
  <button class="ro-btn ro-btn--soft" data-ro-disclosure-trigger aria-haspopup="menu" aria-controls="ref-menu" aria-expanded="false">Opciones</button>
  <div class="ro-menu" id="ref-menu" data-ro-disclosure-panel role="menu" aria-label="Opciones" hidden>
    <button class="ro-menu__item" role="menuitem">Editar</button>
    <button class="ro-menu__item" role="menuitem">Duplicar</button>
    <div class="ro-menu__sep" role="presentation"></div>
    <button class="ro-menu__item ro-menu__item--danger" role="menuitem">Eliminar</button>
  </div>
</div>

Do / Don't

Sí

  • Da nombre al menú (aria-label) y roles a sus ítems.
  • Agrupa con rótulos y separadores cuando ayude.

No

  • No metas navegación del sitio en un role=menu.
  • No mezcles contenido no accionable entre los ítems.

Madurez

experimental Apariencia o API sin garantías completas; puede cambiar sin aviso.
