Tooltip

experimental

Etiqueta contextual breve que aparece al enfocar o pasar el puntero sobre su disparador.

Cuándo usarlo

  • Aclarar la función de un control icónico.
  • Dar una pista corta sin ocupar espacio permanente.

Cuándo no

  • Para contenido interactivo o extenso: usa Popover.
  • Como único medio de dar información esencial: no es accesible en táctil.

Anatomía

<span class="ro-tooltip-trigger">
  <button class="ro-btn-icon" aria-label="Ajustes" aria-describedby="tt-ajustes">
    <svg class="ro-icon" aria-hidden="true"><use href="#ro-i-settings"></use></svg>
  </button>
  <span class="ro-tooltip" id="tt-ajustes" role="tooltip">Ajustes</span>
</span>
  • El disparador referencia el tooltip con aria-describedby.
  • El tooltip lleva role=tooltip; se muestra en :hover y :focus-within (sin JS).
  • El disparador debe ser enfocable para alcanzarlo por teclado.

API

Clases

ClaseDescripción
.ro-tooltip-triggerEnvoltorio posicionador del disparador.
.ro-tooltipBurbuja de la etiqueta (role=tooltip).

Tokens que consume

--ro-ink --ro-on-dark --ro-radius-lg --ro-shadow-md --ro-z-overlay

Sin JavaScript: es CSS sobre HTML nativo.

Teclado y accesibilidad

TeclaAcción
TabEnfoca el disparador y muestra el tooltip.
EscapeOculta el tooltip (recomendado) sin mover el foco.

Contrato de accesibilidad: menu-popover-tooltip

  • Se asocia con aria-describedby, no reemplaza el nombre accesible del control.
  • No pongas foco ni interacción dentro del tooltip.
  • En táctil el tooltip no aparece: no lo uses para información imprescindible.

Ejemplos

Botón icónico con tooltip

Do / Don't

Sí

  • Asócialo con aria-describedby a un control enfocable.
  • Mantén el texto en una línea corta.

No

  • No metas enlaces ni botones dentro del tooltip.
  • No lo uses como único canal de información esencial.

Madurez

experimental Apariencia o API sin garantías completas; puede cambiar sin aviso.

Criterios de promoción en el índice de referencia. Cambios incompatibles en la guía de migración.
