Acciones

Button group

experimental

Agrupa acciones relacionadas en un bloque continuo, o combina una acción con su menú (split).

Cuándo usarlo

  • Ofrecer varias acciones relacionadas del mismo nivel juntas.
  • Combinar una acción principal con opciones secundarias (split button).

Cuándo no

  • Para elegir una opción de un conjunto excluyente: usa Segmented.
  • Para acciones no relacionadas: sepáralas en la Toolbar.

Anatomía

<div class="ro-btn-group">
  <button class="ro-btn ro-btn--soft">Editar</button>
  <button class="ro-btn ro-btn--soft">Duplicar</button>
  <button class="ro-btn ro-btn--soft">Archivar</button>
</div>
  • Cada acción sigue siendo un <button> independiente y enfocable.
  • El split (ro-split) une una acción con un toggle que abre su menú.
  • Si el grupo representa navegación, envuélvelo en un <nav> con nombre.

API

Clases

ClaseDescripción
.ro-btn-groupContenedor que une los botones en un bloque.
.ro-splitBotón dividido: acción + toggle.
.ro-split__toggleDisparador del menú del split.

Tokens que consume

--ro-radius-full --ro-ink-10

Sin JavaScript: es CSS sobre HTML nativo.

Teclado y accesibilidad

TeclaAcción
Tab / Shift+TabRecorre los botones del grupo.
Enter / EspacioActiva el botón enfocado.

Contrato de accesibilidad: native-controls

  • Son botones nativos: no se necesita role adicional.
  • El toggle del split usa aria-haspopup y aria-expanded sobre su menú.

Do / Don't

Sí

  • Agrupa solo acciones del mismo contexto.
  • Mantén etiquetas cortas y paralelas.

No

  • No mezcles acciones destructivas y neutras sin distinción.
  • No uses el grupo para selección excluyente.
