# Principios de producto e ingenieria

1. **Accesibilidad por defecto.** La experiencia por teclado y tecnologias de
   asistencia forma parte de la API, no es una mejora posterior.
2. **Una fuente de verdad.** Los derivados se generan y se verifican.
3. **Contratos antes que demos.** Cada componente documenta estructura,
   comportamiento, estados y limites.
4. **Adopcion gradual.** Importar RoUI no debe alterar interfaces ajenas ni
   obligar a adoptar el sistema completo.
5. **Composicion antes que variantes infinitas.** Las APIs deben ser pequenas,
   consistentes y extensibles.
6. **Calidad verificable.** Una afirmacion de calidad debe tener una prueba o
   criterio reproducible.
7. **Estabilidad con salida.** Toda deprecacion incluye reemplazo, plazo y guia
   de migracion.
8. **Necesidad antes que inventario.** Los componentes se priorizan por casos
   reales y no por cantidad.
9. **Multimarca mediante tokens.** Las marcas cambian decisiones, no bifurcan
   implementaciones.
10. **Documentacion como producto.** Ejemplos y referencias se mantienen con la
    misma exigencia que el codigo publicado.
